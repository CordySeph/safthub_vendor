# วิธีการใช้งาน API: แดชบอร์ดอนุมัติคำขอ (Approval Dashboard)

## ภาพรวม

ระบบนี้ช่วยให้ผู้ดูแลระบบ (Admin) สามารถดูรายการคำขอที่รอการอนุมัติทั้งหมด (เช่น ร้านค้าใหม่, ไรเดอร์ใหม่) และจัดการอนุมัติหรือปฏิเสธคำขอเหล่านั้นได้จากส่วนกลาง

## การยืนยันตัวตน (Authentication)

ทุก API endpoint ที่จะกล่าวถึงด้านล่างนี้ จำเป็นต้องเรียกใช้ในฐานะผู้ใช้ที่เป็นผู้ดูแลระบบ (Admin) และต้องมีสิทธิ์ `applications:manage` โดยต้องแนบ `Authorization` header พร้อมกับ JWT token มาด้วย

---

## Admin API - สำหรับผู้ดูแลระบบ

Base URL: `/api/admin/applications`

### 1.1 ดูรายการคำขอที่รอดำเนินการทั้งหมด

*   **Endpoint:** `GET /pending`
*   **Method:** `GET`
*   **Query Parameters (Optional):**
    *   `page`: ระบุหน้า (e.g., `?page=1`)
    *   `limit`: ระบุจำนวนรายการต่อหน้า (e.g., `?limit=10`)
    *   `type`: กรองตามประเภทของคำขอ (e.g., `?type=restaurant` หรือ `?type=rider`)
*   **การตอบกลับ (Success):** `200 OK` พร้อมรายการคำขอทั้งหมดในรูปแบบ Array และข้อมูล pagination

    ```json
    {
        "data": [
            {
                "id": "uuid-ของ-ร้านค้า",
                "type": "restaurant",
                "name": "ชื่อร้านอาหาร",
                "submitted_at": "2023-01-15T10:00:00Z"
            },
            {
                "id": "uuid-ของ-ไรเดอร์",
                "type": "rider",
                "name": "ชื่อไรเดอร์",
                "submitted_at": "2023-01-16T11:30:00Z"
            }
        ],
        "total": 2,
        "page": 1,
        "limit": 10
    }
    ```

### 1.2 ดูรายละเอียดคำขอร้านค้าที่รอดำเนินการ

*   **Endpoint:** `GET /restaurants/{id}/details`
*   **Method:** `GET`
*   `{id}`: ID (UUID) ของร้านอาหารที่รอดำเนินการ
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อมูลรายละเอียดทั้งหมดของร้านอาหารนั้นๆ (รวมถึงสถานะที่ควรเป็น `pending`)

### 1.3 อนุมัติคำขอร้านค้า

*   **Endpoint:** `POST /restaurants/{id}/approve`
*   **Method:** `POST`
*   `{id}`: ID (UUID) ของร้านอาหารที่ต้องการอนุมัติ
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อความยืนยันและข้อมูลร้านอาหารที่ได้รับการอนุมัติแล้ว

### 1.4 ปฏิเสธคำขอร้านค้า

*   **Endpoint:** `POST /restaurants/{id}/reject`
*   **Method:** `POST`
*   `{id}`: ID (UUID) ของร้านอาหารที่ต้องการปฏิเสธ
*   **Body (JSON):**
    ```json
    {
        "reason": "เอกสารไม่ครบถ้วน" // เหตุผลในการปฏิเสธ (Optional)
    }
    ```
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อความยืนยันและข้อมูลร้านอาหารที่ถูกปฏิเสธ

### 1.5 ดูรายละเอียดคำขอไรเดอร์ที่รอดำเนินการ

*   **Endpoint:** `GET /riders/{id}/details`
*   **Method:** `GET`
*   `{id}`: ID (UUID) ของไรเดอร์ที่รอดำเนินการ
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อมูลรายละเอียดทั้งหมดของไรเดอร์นั้นๆ (รวมถึง `is_approved` ที่ควรเป็น `false`)

### 1.6 อนุมัติคำขอไรเดอร์

*   **Endpoint:** `POST /riders/{id}/approve`
*   **Method:** `POST`
*   `{id}`: ID (UUID) ของไรเดอร์ที่ต้องการอนุมัติ
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อความยืนยันและข้อมูลไรเดอร์ที่ได้รับการอนุมัติแล้ว

### 1.7 ปฏิเสธคำขอไรเดอร์

*   **Endpoint:** `POST /riders/{id}/reject`
*   **Method:** `POST`
*   `{id}`: ID (UUID) ของไรเดอร์ที่ต้องการปฏิเสธ
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อความยืนยัน (หมายเหตุ: ปัจจุบันการปฏิเสธไรเดอร์คือการลบข้อมูลไรเดอร์นั้นออกจากระบบ)
