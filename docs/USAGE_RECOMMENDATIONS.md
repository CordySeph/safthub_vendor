# Recommendation System Usage Guide

This document provides instructions on how to use the recommendation system API to retrieve personalized and popular restaurant recommendations.

## Overview

The recommendation system provides two primary types of restaurant recommendations:
1.  **For You**: Personalized recommendations based on the user's past order history and preferences. These recommendations are pre-calculated by a background job to ensure quick retrieval.
2.  **Popular**: Recommendations for generally popular restaurants across the platform.

## API Endpoint

Recommendations can be accessed via the following API endpoint:

`GET /api/v1/user/recommendations`

### Authentication

This endpoint requires authentication. You must provide a valid JWT token in the `Authorization` header as a Bearer token.

### Query Parameters

| Parameter | Type   | Description                                                                                                                                      | Default     |
| :-------- | :----- | :----------------------------------------------------------------------------------------------------------------------------------------------- | :---------- |
| `type`    | `string` | Specifies the type of recommendations to retrieve. Valid values are `for_you` for personalized recommendations or `popular` for popular recommendations. | `for_you` |

## Examples

### 1. Get "For You" Recommendations

To retrieve personalized recommendations for the authenticated user:

```bash
curl -X GET "http://localhost:8080/api/v1/user/recommendations?type=for_you" 
     -H "Authorization: Bearer <YOUR_JWT_TOKEN>" 
     -H "Content-Type: application/json"
```

### 2. Get Popular Recommendations

To retrieve popular recommendations (no user-specific filtering, but can be location-aware in future versions):

```bash
curl -X GET "http://localhost:8080/api/v1/user/recommendations?type=popular" 
     -H "Authorization: Bearer <YOUR_JWT_TOKEN>" 
     -H "Content-Type: application/json"
```

**Response Example (200 OK):**

(Similar to "For You" recommendations, but the list will be based on overall popularity metrics.)

## Behind the Scenes

The "For You" recommendations are generated by a background job that runs periodically (e.g., every 24 hours). This job analyzes user order history and preferences, stores the results in the database, and ensures that the API can retrieve recommendations quickly without heavy real-time computation. If a user has no order history, the "For You" recommendations might return an empty list or fall back to general popular items.

The "Popular" recommendations are generated by the service in real-time, based on recent order activity and restaurant ratings.
