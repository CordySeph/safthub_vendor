# วิธีการใช้งาน API: การจัดการรีวิว (Review Moderation)

## ภาพรวม

ระบบการจัดการรีวิวช่วยให้ผู้ดูแลระบบ (Admin) สามารถดูรีวิวทั้งหมด, ซ่อนรีวิวที่ไม่เหมาะสมจากสาธารณะ หรือลบรีวิวออกจากระบบได้อย่างถาวร

## การยืนยันตัวตน (Authentication)

ทุก API endpoint ที่จะกล่าวถึงด้านล่างนี้ จำเป็นต้องเรียกใช้ในฐานะผู้ใช้ที่เป็นผู้ดูแลระบบ (Admin) และต้องมีสิทธิ์ `reviews:moderate` โดยต้องแนบ `Authorization` header พร้อมกับ JWT token มาด้วย

---

## Admin API - สำหรับผู้ดูแลระบบ

Base URL: `/api/admin/reviews`

### 1.1 ดูรายการรีวิวทั้งหมด

*   **Endpoint:** `GET /`
*   **Method:** `GET`
*   **Query Parameters (Optional):**
    *   `is_hidden`: กรองตามสถานะการซ่อน (e.g., `?is_hidden=true`)
    *   `page`: ระบุหน้า (e.g., `?page=1`)
    *   `limit`: ระบุจำนวนรายการต่อหน้า (e.g., `?limit=10`)
*   **การตอบกลับ (Success):** `200 OK` พร้อมรายการรีวิวทั้งหมดในรูปแบบ Array และข้อมูล pagination

### 1.2 ดูรายละเอียดรีวิวตาม ID

*   **Endpoint:** `GET /{id}`
*   **Method:** `GET`
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อมูลของรีวิวชิ้นนั้นๆ

### 1.3 อัปเดตสถานะการมองเห็นของรีวิว (ซ่อน/เลิกซ่อน)

*   **Endpoint:** `PATCH /{id}/visibility`
    *   `{id}` คือ ID ของรีวิวที่ต้องการอัปเดต
*   **Method:** `PATCH`
*   **Body (JSON):**
    ```json
    {
        "is_hidden": true // `true` สำหรับซ่อน, `false` สำหรับเลิกซ่อน
    }
    ```
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อมูลของรีวิวที่อัปเดตแล้ว

### 1.4 ลบรีวิวอย่างถาวร

*   **Endpoint:** `DELETE /{id}`
    *   `{id}` คือ ID ของรีวิวที่ต้องการลบ
*   **Method:** `DELETE`
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อความยืนยัน
*   **ข้อควรระวัง:** การดำเนินการนี้เป็นการลบข้อมูลอย่างถาวร (hard delete) ควรใช้ด้วยความระมัดระวัง
