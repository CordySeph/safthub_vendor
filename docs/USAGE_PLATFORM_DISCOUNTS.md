# วิธีการใช้งาน API: การจัดการโปรโมชันและส่วนลดระดับแพลตฟอร์ม (Platform-wide Discount & Promotion Management)

## ภาพรวม

ระบบนี้ช่วยให้ผู้ดูแลระบบ (Admin) สามารถสร้าง, จัดการ, และติดตามโปรโมชันหรือส่วนลดที่ใช้ได้กับทั้งแพลตฟอร์ม เพื่อสนับสนุนแคมเปญการตลาดในวงกว้าง

## การยืนยันตัวตน (Authentication)

ทุก API endpoint ที่จะกล่าวถึงด้านล่างนี้ จำเป็นต้องเรียกใช้ในฐานะผู้ใช้ที่เป็นผู้ดูแลระบบ (Admin) และต้องมีสิทธิ์ `platform-discounts:manage` โดยต้องแนบ `Authorization` header พร้อมกับ JWT token มาด้วย

---

## Admin API - สำหรับผู้ดูแลระบบ

Base URL: `/api/admin/platform-discounts`

### 1.1 สร้างโปรโมชัน/ส่วนลดใหม่ระดับแพลตฟอร์ม

*   **Endpoint:** `POST /`
*   **Method:** `POST`
*   **Body (JSON):**
    ```json
    {
        "code": "WELCOME100",           // Optional: รหัสส่วนลด
        "type": "fixed_amount",         // "percentage", "fixed_amount", "free_delivery"
        "value": 100.00,                // มูลค่าส่วนลด (เช่น 100 สำหรับ 100 บาท หรือ 10.0 สำหรับ 10%)
        "min_order_value": 500.00,      // ยอดสั่งซื้อขั้นต่ำ
        "max_discount_amount": null,    // Optional: เพดานส่วนลดสูงสุดสำหรับ percentage
        "start_date": "2024-03-01T00:00:00Z",
        "end_date": "2024-03-31T23:59:59Z",
        "usage_limit": 1000,            // 0 = ไม่จำกัดจำนวนการใช้งานทั้งหมด
        "per_user_limit": 1,            // 0 = ไม่จำกัดจำนวนการใช้งานต่อผู้ใช้
        "is_active": true,
        "applies_to_new_users": false   // true หากใช้ได้เฉพาะผู้ใช้ใหม่
    }
    ```
*   **การตอบกลับ (Success):** `201 Created` พร้อมข้อมูลของโปรโมชัน/ส่วนลดที่สร้างขึ้น

### 1.2 ดูรายการโปรโมชัน/ส่วนลดทั้งหมดระดับแพลตฟอร์ม

*   **Endpoint:** `GET /`
*   **Method:** `GET`
*   **Query Parameters (Optional):**
    *   `page`: ระบุหน้า (e.g., `?page=1`)
    *   `limit`: ระบุจำนวนรายการต่อหน้า (e.g., `?limit=10`)
    *   `is_active`: กรองตามสถานะการใช้งาน (e.g., `?is_active=true`)
    *   `type`: กรองตามประเภทส่วนลด (e.g., `?type=percentage`)
*   **การตอบกลับ (Success):** `200 OK` พร้อมรายการโปรโมชัน/ส่วนลดทั้งหมดในรูปแบบ Array และข้อมูล pagination

### 1.3 ดูรายละเอียดโปรโมชัน/ส่วนลดระดับแพลตฟอร์ม

*   **Endpoint:** `GET /{id}`
*   **Method:** `GET`
*   `{id}`: ID (UUID) ของโปรโมชัน/ส่วนลด
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อมูลรายละเอียดของโปรโมชัน/ส่วนลดนั้นๆ

### 1.4 อัปเดตโปรโมชัน/ส่วนลดระดับแพลตฟอร์ม

*   **Endpoint:** `PUT /{id}`
*   **Method:** `PUT`
*   `{id}`: ID (UUID) ของโปรโมชัน/ส่วนลด
*   **Body (JSON):** เหมือนกับตอนสร้างโปรโมชัน/ส่วนลด (`POST /`) แต่ต้องมี `id` ใน URL
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อมูลของโปรโมชัน/ส่วนลดที่อัปเดตแล้ว

### 1.5 ลบโปรโมชัน/ส่วนลดระดับแพลตฟอร์ม

*   **Endpoint:** `DELETE /{id}`
*   **Method:** `DELETE`
*   `{id}`: ID (UUID) ของโปรโมชัน/ส่วนลด
*   **การตอบกลับ (Success):** `200 OK` พร้อมข้อความยืนยันการลบ
